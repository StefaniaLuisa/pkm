Recommendations for the on-going work for this project:

**Review and Address Technical Issues**

- Use Google Search Console and check the “Pages” and “Coverage” reports for errors and warnings (e.g., 404s, redirects, crawl issues).

- Prioritize fixing critical errors first (such as 404 pages and setup redirects).

  - Redirects are setup in WP Plugin called Redirects - found in sidebar of WP Dashboard

- For each error, click for details and use the provided URLs to identify the source on SCAN’s website.

- Document issues in a shared spreadsheet or task tracker, noting the URL, error type, and recommended fix. If it is something that be fixed by redirecting or something else immediate - please still document and make update, then reindex the page and note when the reindex was submitted in the doc.

**Continue work on SCAN’s website using Yoast**

- For each new or updated page, review the Yoast SEO analysis:

  - Document and update the SEO title and meta description to include primary keywords and a clear call to action.

  - Check the readability analysis and address any flagged issues (shorten sentences, add headings, etc.).

  - Make sure there is a featured image and complete the social sharing fields if not already synced up with the title & meta tags

- For blog posts, resources, news etc, ensure there is a clear CTA internal links to other relevant SCAN content. Also, if opportunity to internal linking is needed - recommend.

- Ensure schema markup is enabled through Yoast - default is Article which covers most instances but sometimes there are more relevant schema options

  - If relevant - blogs/news/resources - all should have a CTA to read more stories, subscribe, take action etc.

**Ongoing Monitoring and Reporting**

- Use Google Search Console’s “Performance” report to identify pages with high impressions but low click-through rates (CTR). Review and update titles/meta descriptions for those pages to be more compelling. Document the original and uthe updated.

- After making changes, use the “URL Inspection” tool in Search Console to request indexing.

- Track progress in a shared document, note changes made, issues resolved, and next priorities.

404 Redirect Process for SCAN

1. Identify pages that get significant organic traffic by pulling the Organic Traffic report in Google Analytics for the past year or so.

2. Identify pages that have a significant number of links (or, have links from high traffic sources) in Open Site Explorer.

3. Map where that content should be now, and 301 redirect to new pages.

4. Completely remove all old pages from the index by 404ing them and making sure that no links on new pages point to old pages.

Sounds quick and simple, but this definitely takes time. Good luck!
