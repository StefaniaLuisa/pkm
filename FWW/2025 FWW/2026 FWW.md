<https://app.productive.io/2650-4site-interactive-studios-inc/tasks/12890296>

Hi Fernando,

Bryan and I met to review what might be happening with the digital wallets not showing.

Here’s what Bryan and I are seeing:

1. **Desktop behavior**

   - If we load the donation page directly (no lightbox), digital wallet buttons display as expected.

   - If we trigger the **multi-step lightbox** and simply resize the desktop browser window down below the mobile breakpoint, the digital wallet buttons still show.

   - If we instead use **Chrome DevTools’ Inspector Tools** to preview a mobile viewport, the digital wallet buttons do **not** show in the multi-step lightbox, at all

2. **Actual mobile behavior (iPhone)**

   - On an actual iPhone, digital wallet buttons (Apple Pay / Google Pay) **do not show** in the **multi-step lightbox**.

   - If we load the same donation form page directly (the iFramed page the lightbox uses), the digital wallet buttons **do** appear.

   - So the wallets work on the base form itself, but disappear in the context of the multi-step overlay.

3. **Breakpoint / layout weirdness**

   - As the layout crosses the mobile breakpoint, or gets close to that breakpoint there is this “in-between” range of widths where the digital wallets disappear

   - Bryan checked, the buttons exist in the DOM but are not rendering correctly within the multi-step flow.

4. 

Bryan also wanted me to lt you know that his hunch may be related to an issue you mentioned recently to him. That it is something to do with the hide the wallet buttons on load within the multi-step lightbox. That is can cause the detection logic to think digital wallets are not enabled, which then prevents them from rendering even though Stripe is configured correctly.

I can send you the exact embedded page URL and the FWW multi-step QA URL we’re using for testing if that helps, but I wanted to get you the behavioral summary first.
