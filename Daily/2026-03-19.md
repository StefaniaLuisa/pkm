**The Root Cause of the Page View Drop**

- In the version of GTM that was live before March 10 (V799 and earlier), there were two overlapping Google Tags both configured to fire on every page:

1. **"GA4 – Configuration"** tag with `send_page_view: true` firing on every page initialization

2. **"GA4 Event – All Pages – NWF.org"** tag firing on both an "All Page Views" trigger AND an "Initialization – All Pages" trigger simultaneously

- **This caused every single page visit to register \~2-3 page_view events in GA4, artificially inflating the count to approximately 100K.**

- When NWF published Version 800 on March 10, **the duplicate "GA4 Event – All Pages – NWF.org" was correctly paused.** This removed the inflated counting and brought the number down to \~60K — which is actually **the correct, accurate number**.

**What Happened Today (March 19)**

- NWF rolled back the container to Version 799 this morning, which re-activated the duplicate/triple-counting tag and also removed the PII exclusion that was protecting password reset pages from GA4 tracking. **The site is currently over-counting again and the PII protection is off.**
