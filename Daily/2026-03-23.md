## Edge Cases & Expected Behavior

### Non-PaintCare States

- **Examples:** `Houston TX`, `Miami FL`, `77001`
- **Current:** Search works, but results include third-party HHW sites — not PaintCare partners. The disclaimer *"[State] is not a PaintCare State."* is unstyled and easy to miss.
- **Fix:** add a yellow ⚠️ icon with the same yellow pill tag with bold text. Add context: *"These are third-party household hazardous waste facilities, not PaintCare drop-off partners."*

### Gibberish / Unrecognizable Input

- **Examples:** `xqzwbvf`, `aaaaaaaaa`
- **Current:** Dropdown shows "No results found." with no guidance.

---

### Special Characters & Symbols

- **Examples:** `!@#$%`, `<script>`
- **Current:** Geocoder throws an internal error; user sees *"There was an error reaching the server."* which is misleading.

---

### Latitude / Longitude Coordinates

- **Examples:** `40.7128,-74.0060`
- **Current:** API returns HTTP `422`. User sees the generic server error.
- **Fix:** Detect lat/lon pattern with regex **before** the API call fires. Either silently reverse-geocode to the nearest address, or display: *"Coordinates aren't supported. Please enter a city, ZIP code, or street address."*

**IDK, this one doesn’t seem like a bad thing to allow - but if it doesn’t work automatically I would just throw the error language.**

---

### Business / Point-of-Interest Names

- **Examples:** `Home Depot`, `Walmart`, `Sherwin-Williams`
- **Current:** `poi` type is excluded from config. API returns streets with matching words; geocoder rejects and shows generic server error.
- **Fix:** Add a persistent hint below the input: *"Search by location, not business name."*

### International Place Names

- **Examples:** `London`, `Paris`, `Tokyo`
- **Current:** Search is restricted to `country=US` silently. Users get US cities with matching names (London, KY; Paris, TX) with no indication the field is US-only.
- **Fix:** Add a label near the input: *"United States locations only.*

### Partial ZIP Codes

- **Examples:** `90`, `441`
- **Current:** API returns up to five matching ZIPs. Functional, but suggestions may feel geographically random.
- **Fix:** No change required. Optional: bias suggestions toward user's region if browser geolocation is available.-

---

### 8. Natural Language Queries

**Examples:** `paint store near me`, `where can I recycle`

**Current:** No geographic match found; geocoder throws internal error and shows generic server error.

**Fix:** Catch the error and show the invalid input message (see #2). Add hint text below the input: *"Examples: 90210 · Portland OR · 123 Main St Seattle"*

---

### 9. Single-Character Input

**Current:** No API call fires (correct), but stale error messages from the previous query remain visible in the dropdown.

**Fix:** Clear all dropdown content and errors the moment input drops below 2 characters.
